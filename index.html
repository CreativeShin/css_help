Animations In CSS

Keyframes and Transition

Keyframes Here!

  1. Browsers are well optimized for transform and opacity in keyframes.
  2. Backface visibility - hidden in the parent is used to fix the shakes.
  3. We can add animations on events and not just at page load.
  4. Keyframes: 
        @keyframes animationName{
        0%{
            
        }
        100%{

        }
    }
  5. The way we add animation to an element: 
        p{
        animation-name: animationName;
        animation-duration: 1s;
    }
  6. Other properties: 
        p{
        animation-timing-function: ease-in;
        animation-delay: 2s;

    }
  7. Ex Definition : 
            @keyframes genericAnimation{
            0%{
                transform: translate(-100px);
                opacity: 0;
            }
            100%{
                transform: translate(0);
                opacity: 1;
            }
        }
  8. Example: 
            .example-keyframes{
            padding:10px;
            font-size: 25px;
            animation-name: genericAnimation;
            animation-duration: 5s;
        }

Some CSS Properties

Pseudo Class and Pseudo Elements

  1. Pseudo Classes are used to select elements in special conditions or state. 
        :link
    :visited 
    :hover
    :active
  2. UI Pseudo Classes: 
        :enabled
    :disabled
    :checked
  3. Pseudo Elements allow us to style certain parts of element. 
        ::before
    ::after
    ::first-letter
    ::first-line
  4. After pseudo element adds a virtual element.
  5.     
    .btn::after{
        content: "";
        display: inline-block; /*As the button is inline block*/
        height: 100%; /*Treated as a child*/
        width: 100%;
        border-radius: 100px;
        position: absolute;
        top:0;
        left:0;
        z-index: -1;
        transition: all 0.4s;
    }
  6. Note: animation-fill-mode: backwards;Applies the styles of 0% before the animation starts. It is used when animation-delay is there.

Definitions

  1. Cascade is the process of combining different stylesheets and resolving conflicts between different CSS rules and declarations, when more than one rule applies to a certain element.
  2. CSS declarations marked with !important have the highest priority.
  3. We should use !important as the last resource, it is better to use correct specificity. More maintainable code.
  4. Inline styles will always have priority over styles in external stylesheets.
  5. A selector that contains one ID is more specific than one with 1000 classes.
  6. A selector with one class is more specific than one with 1000 elements.
  7. The universal selector (*) has no specificity.
  8. We should rely more on specificity than order of the selectors.
  9. We should always put author stylesheets last when using 3rd party stylesheets.
